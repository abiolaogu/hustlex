### Technology Choices

| Layer | Technology | Rationale |
|-------|------------|-----------|
| **Mobile** | Flutter | Cross-platform, fast development, good performance |
| **Backend** | Go (Fiber) | High performance, strong typing, great for fintech |
| **Database** | PostgreSQL | Reliable, ACID compliant, good for financial data |
| **Cache** | Redis | Fast, versatile, handles sessions and queues |
| **Jobs** | Asynq | Redis-based, Go-native, reliable job processing |
| **Search** | PostgreSQL FTS | Simple, integrated, sufficient for now |
| **Cloud** | AWS / GCP | Reliable, scalable, Nigeria regions available |
| **Monitoring** | Prometheus + Grafana | Industry standard, open source |
| **Logging** | ELK / CloudWatch | Centralized logging, searchable |
| **CI/CD** | GitHub Actions | Integrated, free for public repos |

### Security Architecture

| Layer | Measures |
|-------|----------|
| **Network** | TLS 1.3, DDoS protection, WAF |
| **Application** | Input validation, parameterized queries, rate limiting |
| **Authentication** | JWT, OTP, PIN, optional biometrics |
| **Data** | AES-256 encryption at rest, field-level encryption for PII |
| **Infrastructure** | VPC isolation, security groups, non-root containers |
| **Monitoring** | Real-time alerts, audit logs, intrusion detection |

### RACI Legend

- **R** = Responsible (does the work)
- **A** = Accountable (owns the outcome)
- **C** = Consulted (provides input)
- **I** = Informed (kept updated)
